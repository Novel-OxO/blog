## HTML이란

HTML은 **HyperText Markup Language**의 약자로, 웹 페이지를 구조화하고 내용을 표현하기 위한 마크업 언어다.

- **HyperText**: 링크를 통해 다른 문서나 콘텐츠로 즉시 이동할 수 있는 텍스트. 기존의 선형적 텍스트와 달리, 사용자가 원하는 순서로 정보를 탐색할 수 있다.
- **Markup**: 텍스트에 태그를 붙여 구조와 의미를 부여하는 것. 어떤 텍스트가 제목인지, 단락인지, 링크인지를 태그로 표시하여 브라우저가 올바르게 해석하고 렌더링할 수 있도록 한다.

## 태그

태그는 HTML의 기본 구성 요소로, 꺾쇠괄호(`< >`)로 감싸서 표현한다.

- **여는 태그**: `<태그명>` — 요소의 시작
- **닫는 태그**: `</태그명>` — 요소의 끝
- **빈 태그(self-closing)**: `<br>`, `<img>`, `<input>` 등 닫는 태그 없이 단독으로 사용

```html
<a href="https://example.com">링크 텍스트</a>
```

| 구성 요소     | 설명                                      |
| ------------- | ----------------------------------------- |
| `a`           | 태그명 (요소 이름)                        |
| `href="..."`  | 속성(attribute) — 태그에 추가 정보를 부여 |
| `링크 텍스트` | 콘텐츠 — 태그 사이에 들어가는 내용        |

### 부모-자식 관계

HTML 태그는 다른 태그 안에 중첩(nesting)될 수 있으며, 이를 통해 트리 구조를 형성한다.

```html
<div>
  <h1>제목</h1>
  <p>단락</p>
</div>
```

| 용어             | 설명                                        |
| ---------------- | ------------------------------------------- |
| 부모(parent)     | 다른 요소를 감싸고 있는 바깥쪽 요소 (`div`) |
| 자식(child)      | 부모 안에 직접 포함된 요소 (`h1`, `p`)      |
| 형제(sibling)    | 같은 부모를 공유하는 요소 (`h1`과 `p`)      |
| 자손(descendant) | 부모 안에 포함된 모든 하위 요소             |

이 계층 구조를 **DOM(Document Object Model) 트리**라고 하며, CSS 선택자나 JavaScript에서 요소를 탐색할 때 기반이 된다.

## 기본 문서 구조

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body></body>
</html>
```

| 코드                     | 설명                                                                              |
| ------------------------ | --------------------------------------------------------------------------------- |
| `<!DOCTYPE html>`        | 이 문서가 HTML5임을 브라우저에 선언한다. 태그가 아닌 선언문이다.                  |
| `<html lang="en">`       | HTML 문서의 루트(최상위) 요소. `lang` 속성은 문서의 주 언어를 지정한다.           |
| `<head>`                 | 브라우저에 표시되지 않는 메타 정보를 담는 영역이다.                               |
| `<meta charset="UTF-8">` | 문자 인코딩을 UTF-8로 설정한다. 한글 등 다국어를 올바르게 표시하기 위해 필요하다. |
| `<meta name="viewport">` | 모바일 브라우저의 렌더링 방식을 제어한다. 아래 상세 설명 참고.                    |
| `<title>`                | 브라우저 탭에 표시되는 문서 제목을 지정한다.                                      |
| `<body>`                 | 사용자에게 실제로 보이는 콘텐츠가 들어가는 영역이다.                              |

### viewport 메타 태그

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

모바일 브라우저는 기본적으로 페이지를 데스크탑 너비(보통 980px)로 렌더링한 뒤 축소해서 보여준다. 이 태그를 설정하면 기기의 실제 화면 너비에 맞춰 렌더링한다.

| 속성                 | 설명                                                                                                      |
| -------------------- | --------------------------------------------------------------------------------------------------------- |
| `width=device-width` | 뷰포트 너비를 기기의 화면 너비와 동일하게 설정한다. 이 설정이 없으면 모바일에서 페이지가 축소되어 보인다. |
| `initial-scale=1.0`  | 페이지 로드 시 초기 확대/축소 비율을 1배(100%)로 설정한다.                                                |

이 태그가 없으면 CSS 미디어 쿼리가 의도대로 동작하지 않아 반응형 레이아웃이 깨질 수 있다.

## 텍스트 요소

### 제목 태그

`<h1>` ~ `<h6>` 태그로 제목을 나타내며, 숫자가 작을수록 높은 수준의 제목이다.

```html
<h1>가장 큰 제목</h1>
<h2>두 번째 제목</h2>
<h3>세 번째 제목</h3>
```

- `<h1>`은 페이지당 하나만 사용하는 것이 권장된다 (SEO 및 접근성).
- 단순히 글자 크기를 키우기 위해 사용하면 안 된다. 글자 크기는 CSS로 조절한다.
- 제목 수준을 건너뛰지 않는 것이 좋다 (예: `<h1>` 다음에 바로 `<h3>` 사용 지양).

### 단락과 줄바꿈

```html
<p>첫 번째 단락입니다.</p>
<p>두 번째 단락입니다.</p>

<p>첫 번째 줄<br />두 번째 줄</p>
```

- `<p>`: 텍스트를 하나의 단락으로 묶는다. 단락 사이에는 자동으로 여백이 생긴다.
- `<br>`: 텍스트 내에서 강제로 줄을 바꾼다. HTML에서는 코드 상의 줄바꿈이나 연속 공백이 무시되므로, 줄바꿈이 필요하면 `<br>`을 사용한다.

### 수평선, 인용문, 서식 유지

```html
<!-- 주제 구분 수평선 -->
<hr />

<!-- 인용문 -->
<blockquote>
  <p>배움에는 왕도가 없다.</p>
</blockquote>

<!-- 공백과 줄바꿈을 그대로 표시 -->
<pre>
  이   텍스트는
  그대로    표시됩니다.
</pre>
```

### 텍스트 강조와 서식

| 태그       | 설명                          | 렌더링         |
| ---------- | ----------------------------- | -------------- |
| `<strong>` | 의미적으로 중요한 텍스트      | **굵은 글씨**  |
| `<em>`     | 강조(emphasis)하는 텍스트     | _기울임_       |
| `<b>`      | 시각적으로 굵게 (의미 없음)   | **굵은 글씨**  |
| `<i>`      | 시각적으로 기울임 (의미 없음) | _기울임_       |
| `<del>`    | 삭제된 텍스트                 | ~~취소선~~     |
| `<ins>`    | 추가된 텍스트                 | 밑줄           |
| `<mark>`   | 하이라이트                    | 형광펜 효과    |
| `<sub>`    | 아래첨자                      | H₂O            |
| `<sup>`    | 위첨자                        | X²             |
| `<code>`   | 인라인 코드                   | 고정폭 글꼴    |
| `<span>`   | 인라인 범용 컨테이너          | 스타일 적용 용 |

> `<strong>`과 `<b>`는 모두 굵게 보이지만, `<strong>`은 의미적 중요성을 나타내고 `<b>`는 단순 시각적 효과다. 스크린 리더는 `<strong>`을 강조하여 읽는다. `<em>`과 `<i>`도 같은 차이가 있다.

```html
<p>가격: <del>10,000원</del> <ins>8,000원</ins></p>
<p>물의 화학식은 H<sub>2</sub>O이다.</p>
<p><code>console.log()</code>로 값을 확인한다.</p>
```

## 컨테이너 요소

### `<div>` 태그

`<div>`는 Division(구분)의 약자로, 콘텐츠를 논리적으로 그룹화하기 위한 범용 컨테이너다. 자체적으로는 아무런 의미나 스타일이 없다.

```html
<div class="header">헤더 영역</div>
<div class="content">본문 영역</div>
<div class="footer">푸터 영역</div>
```

주요 용도

- **레이아웃 구성**: CSS와 함께 사용하여 페이지 영역을 나눈다.
- **스타일 적용**: `class`나 `id`를 부여해 CSS 스타일을 일괄 적용한다.
- **JavaScript 조작**: 여러 요소를 하나로 묶어 DOM 조작을 쉽게 한다.

> `<div>`는 의미가 없는 태그이므로, 의미를 가진 시맨틱 태그가 적합한 경우에는 시맨틱 태그를 우선 사용하는 것이 좋다.

## 목록 태그

### 순서 없는 목록 / 순서 있는 목록

```html
<!-- 순서 없는 목록 — 점(bullet) 표시 -->
<ul>
  <li>사과</li>
  <li>바나나</li>
  <li>포도</li>
</ul>

<!-- 순서 있는 목록 — 번호 자동 매김 -->
<ol>
  <li>회원가입</li>
  <li>로그인</li>
  <li>서비스 이용</li>
</ol>
```

`<li>`는 반드시 `<ul>`이나 `<ol>`의 자식으로 사용해야 한다. 목록 안에 목록을 넣어 계층 구조도 만들 수 있다.

```html
<ul>
  <li>
    프론트엔드
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>백엔드</li>
</ul>
```

### 정의 목록

용어와 그 설명을 짝지어 나열할 때 사용한다.

```html
<dl>
  <dt>HTML</dt>
  <dd>웹 페이지의 구조를 정의하는 마크업 언어</dd>
  <dt>CSS</dt>
  <dd>웹 페이지의 스타일을 지정하는 스타일시트 언어</dd>
</dl>
```

| 태그   | 설명                                     |
| ------ | ---------------------------------------- |
| `<dl>` | 정의 목록 전체를 감싸는 컨테이너         |
| `<dt>` | 용어(Definition Term)                    |
| `<dd>` | 용어에 대한 설명(Definition Description) |

## 테이블

### 기본 구조

```html
<table>
  <tr>
    <th>이름</th>
    <th>나이</th>
  </tr>
  <tr>
    <td>홍길동</td>
    <td>25</td>
  </tr>
  <tr>
    <td>김철수</td>
    <td>30</td>
  </tr>
</table>
```

| 태그      | 설명                                                      |
| --------- | --------------------------------------------------------- |
| `<table>` | 표 전체를 감싸는 컨테이너                                 |
| `<tr>`    | 행(Table Row)                                             |
| `<th>`    | 헤더 셀(Table Header) — 기본적으로 굵은 글씨, 가운데 정렬 |
| `<td>`    | 데이터 셀(Table Data) — 일반 셀                           |

### 구조화된 테이블

`<thead>`, `<tbody>`, `<tfoot>`으로 머리글, 본문, 바닥글을 구분하여 의미를 명확히 할 수 있다.

```html
<table>
  <thead>
    <tr>
      <th>과목</th>
      <th>점수</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>국어</td>
      <td>90</td>
    </tr>
    <tr>
      <td>수학</td>
      <td>85</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>평균</td>
      <td>87.5</td>
    </tr>
  </tfoot>
</table>
```

### 셀 병합

```html
<td colspan="2">가로 2칸 병합</td>
<td rowspan="3">세로 3칸 병합</td>
```

| 속성      | 설명                              |
| --------- | --------------------------------- |
| `colspan` | 가로 방향으로 셀을 병합 (열 병합) |
| `rowspan` | 세로 방향으로 셀을 병합 (행 병합) |

## 미디어 요소

### 이미지

`<img>` 태그는 웹 페이지에 이미지를 삽입한다. 닫는 태그가 없는 빈 태그다.

```html
<img src="photo.jpg" alt="풍경 사진" />
```

| 속성     | 설명                                                                                       |
| -------- | ------------------------------------------------------------------------------------------ |
| `src`    | 이미지 파일의 경로 (필수)                                                                  |
| `alt`    | 이미지를 표시할 수 없을 때 대체 텍스트 (필수). 스크린 리더가 읽어주므로 접근성에 중요하다. |
| `width`  | 이미지의 너비 (px 단위)                                                                    |
| `height` | 이미지의 높이 (px 단위)                                                                    |

경로는 상대 경로(`images/photo.jpg`), 절대 경로(`/images/photo.jpg`), 외부 URL(`https://...`) 세 가지 방식으로 지정할 수 있다.

### 비디오

```html
<video controls autoplay muted loop poster="thumbnail.jpg" width="640">
  <source src="movie.mp4" type="video/mp4" />
  <source src="movie.webm" type="video/webm" />
  이 브라우저는 비디오 태그를 지원하지 않습니다.
</video>
```

| 속성       | 설명                                                                          |
| ---------- | ----------------------------------------------------------------------------- |
| `controls` | 재생, 일시정지, 볼륨 등 컨트롤 UI를 표시한다                                  |
| `autoplay` | 페이지 로드 시 자동 재생 (대부분의 브라우저에서 `muted`와 함께 사용해야 동작) |
| `muted`    | 음소거 상태로 시작                                                            |
| `loop`     | 영상이 끝나면 자동으로 반복 재생                                              |
| `poster`   | 재생 전에 표시할 썸네일 이미지 경로                                           |

`<source>` 태그로 여러 형식(MP4, WebM, Ogg)을 제공하면 브라우저가 지원하는 첫 번째 형식을 재생한다.

### 오디오

```html
<audio controls>
  <source src="music.mp3" type="audio/mpeg" />
  <source src="music.ogg" type="audio/ogg" />
  이 브라우저는 오디오 태그를 지원하지 않습니다.
</audio>
```

사용법은 `<video>`와 유사하며, `controls`, `autoplay`, `muted`, `loop` 속성을 동일하게 사용할 수 있다.

### iframe

`<iframe>`(Inline Frame)은 현재 페이지 안에 다른 HTML 페이지나 외부 콘텐츠를 삽입한다.

```html
<!-- 유튜브 영상 삽입 -->
<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  title="유튜브 영상"
  allowfullscreen
>
</iframe>
```

| 속성              | 설명                                                |
| ----------------- | --------------------------------------------------- |
| `src`             | 삽입할 페이지의 URL                                 |
| `title`           | iframe 콘텐츠에 대한 설명 (접근성을 위해 필수 권장) |
| `allowfullscreen` | 전체 화면 모드 허용                                 |

> 보안상 많은 사이트는 자신의 페이지가 iframe으로 삽입되는 것을 차단한다. iframe 남용은 페이지 성능 저하의 원인이 될 수 있으므로 필요한 경우에만 사용하는 것이 좋다.

## 링크 태그

`<a>`(Anchor) 태그는 다른 페이지나 리소스로 이동하는 하이퍼링크를 만든다.

```html
<a href="https://example.com">예시 사이트</a>
<a href="https://example.com" target="_blank">새 탭에서 열기</a>
```

| 속성     | 설명                                      |
| -------- | ----------------------------------------- |
| `href`   | 링크 대상의 URL (필수)                    |
| `target` | `_self`(현재 탭, 기본값), `_blank`(새 탭) |
| `title`  | 마우스를 올렸을 때 표시되는 툴팁 텍스트   |

> `target="_blank"` 사용 시 보안을 위해 `rel="noopener noreferrer"`를 함께 추가하는 것이 권장된다.

### 다양한 링크 유형

```html
<!-- 같은 사이트 내 페이지 이동 -->
<a href="/about.html">소개 페이지</a>

<!-- 페이지 내 특정 위치로 이동 (앵커) -->
<a href="#section1">섹션1로 이동</a>
<h2 id="section1">섹션1</h2>

<!-- 이메일 / 전화 링크 -->
<a href="mailto:test@example.com">메일 보내기</a>
<a href="tel:010-1234-5678">전화 걸기</a>
```

## `<figure>` / `<figcaption>`

`<figure>`는 이미지, 도표, 코드 등 독립적인 콘텐츠를 감싸는 시맨틱 태그이며, `<figcaption>`은 캡션을 붙인다.

```html
<figure>
  <img src="photo.jpg" alt="풍경 사진" />
  <figcaption>제주도 해변 풍경</figcaption>
</figure>

<figure>
  <blockquote>배움에는 왕도가 없다.</blockquote>
  <figcaption>— 유클리드</figcaption>
</figure>
```

> `<figure>`는 단순히 이미지를 감싸는 용도가 아니라, 본문에서 참조되는 독립적인 콘텐츠(그림, 표, 코드, 인용문 등)를 의미적으로 묶는 태그다.

## 폼 요소

### 기본 구조

```html
<form action="/submit" method="POST">
  <label for="username">이름</label>
  <input type="text" id="username" name="username" />
  <button type="submit">제출</button>
</form>
```

| 속성     | 설명                                                     |
| -------- | -------------------------------------------------------- |
| `action` | 폼 데이터를 전송할 서버 URL                              |
| `method` | `GET`(URL에 데이터 포함) 또는 `POST`(본문에 데이터 포함) |

### `<input>` 타입

| type       | 설명                                |
| ---------- | ----------------------------------- |
| `text`     | 한 줄 텍스트 입력                   |
| `password` | 비밀번호 입력 (입력 내용이 가려짐)  |
| `email`    | 이메일 형식 입력 (자동 유효성 검사) |
| `number`   | 숫자 입력                           |
| `checkbox` | 체크박스 (다중 선택)                |
| `radio`    | 라디오 버튼 (단일 선택)             |
| `file`     | 파일 업로드                         |
| `hidden`   | 사용자에게 보이지 않는 숨겨진 값    |

```html
<input type="text" name="name" placeholder="이름을 입력하세요" />
<input type="password" name="pw" placeholder="비밀번호" />
<input type="email" name="email" placeholder="이메일" />
<input type="checkbox" name="agree" id="agree" />
<input type="radio" name="gender" value="male" /> 남성
<input type="radio" name="gender" value="female" /> 여성
```

### `<input>` 주요 속성

| 속성          | 설명                                   |
| ------------- | -------------------------------------- |
| `name`        | 서버로 전송될 때 데이터의 키(key) 이름 |
| `value`       | 입력 필드의 기본값                     |
| `placeholder` | 입력 전 안내 텍스트 (입력 시 사라짐)   |
| `required`    | 필수 입력 항목으로 지정                |
| `disabled`    | 입력 비활성화                          |
| `readonly`    | 읽기 전용 (값 수정 불가, 전송은 됨)    |

### `<label>` — 입력 필드 레이블

`for` 속성과 입력 필드의 `id`를 일치시키면 레이블 클릭 시 해당 필드에 포커스된다.

```html
<label for="email">이메일</label> <input type="email" id="email" name="email" />
```

### `<textarea>`, `<select>`

```html
<!-- 여러 줄 텍스트 입력 -->
<textarea name="message" rows="5" cols="30" placeholder="내용을 입력하세요"></textarea>

<!-- 드롭다운 선택 -->
<select name="city">
  <option value="">선택하세요</option>
  <option value="seoul">서울</option>
  <option value="busan">부산</option>
  <option value="jeju" selected>제주</option>
</select>
```

### 버튼

```html
<button type="submit">제출</button>
<button type="button">일반 버튼</button>
<button type="reset">초기화</button>
<button disabled>비활성화된 버튼</button>
```

| type     | 설명                                                 |
| -------- | ---------------------------------------------------- |
| `submit` | 폼 데이터를 제출한다 (기본값, `<form>` 안에 있을 때) |
| `button` | 아무 동작 없음. JavaScript로 기능을 직접 지정한다    |
| `reset`  | 폼 내 입력값을 초기화한다                            |

> `<form>` 안에서 `type`을 지정하지 않으면 기본값이 `submit`이므로, 폼 제출 의도가 없다면 반드시 `type="button"`을 명시해야 한다.

## 시맨틱 HTML

시맨틱 태그는 콘텐츠의 의미와 역할을 명확히 전달하는 태그다. `<div>`로도 동일한 레이아웃을 만들 수 있지만, 시맨틱 태그를 사용하면 코드의 가독성, 접근성, SEO가 향상된다.

### 비시맨틱 vs 시맨틱

```html
<!-- 비시맨틱 -->
<div class="header">...</div>
<div class="nav">...</div>
<div class="content">...</div>
<div class="footer">...</div>

<!-- 시맨틱 -->
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>
```

### 주요 시맨틱 태그

| 태그        | 설명                                                                |
| ----------- | ------------------------------------------------------------------- |
| `<header>`  | 페이지나 섹션의 머리글. 로고, 제목, 내비게이션 등을 포함한다        |
| `<nav>`     | 내비게이션 링크 모음. 메뉴, 목차 등에 사용한다                      |
| `<main>`    | 페이지의 핵심 콘텐츠. 페이지당 하나만 사용한다                      |
| `<section>` | 주제별로 그룹화된 콘텐츠 영역. 보통 제목을 포함한다                 |
| `<article>` | 독립적으로 배포·재사용 가능한 콘텐츠. 블로그 글, 뉴스 기사, 댓글 등 |
| `<aside>`   | 본문과 간접적으로 관련된 부가 콘텐츠. 사이드바, 광고, 관련 링크 등  |
| `<footer>`  | 페이지나 섹션의 바닥글. 저작권, 연락처, 사이트맵 등을 포함한다      |

### 레이아웃 예시

```html
<header>
  <h1>사이트 제목</h1>
  <nav>
    <a href="/">홈</a>
    <a href="/about">소개</a>
    <a href="/contact">연락처</a>
  </nav>
</header>

<main>
  <section>
    <h2>최신 글</h2>
    <article>
      <h3>글 제목</h3>
      <p>글 내용...</p>
    </article>
  </section>

  <aside>
    <h3>관련 링크</h3>
    <ul>
      <li><a href="#">링크 1</a></li>
      <li><a href="#">링크 2</a></li>
    </ul>
  </aside>
</main>

<footer>
  <p>© 2026 All rights reserved.</p>
</footer>
```

### 시맨틱 태그를 사용하는 이유

- **접근성**: 스크린 리더가 페이지 구조를 파악하여 시각 장애인에게 내비게이션을 제공한다.
- **SEO**: 검색 엔진이 콘텐츠의 중요도와 구조를 이해하여 검색 결과에 반영한다.
- **가독성**: 개발자가 코드만 보고도 페이지 구조를 쉽게 파악할 수 있다.
